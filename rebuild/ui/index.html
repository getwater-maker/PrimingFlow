📖 사용 설명서

🚀 빠른 시작 (3단계)

  1. Google 로그인 — 계정 선택 후 [로그인] 클릭, 브라우저에서 로그인 완료
  2. 대본 입력 — 텍스트 붙여넣기 또는 파일 열기 (빈 줄로 단락 구분)
  3. 생성 시작 — 출력 폴더 선택 후 [생성 시작] 클릭

📄 대본 파일 (.md) 구조

대본은 일반 텍스트(.txt) 또는 마크다운(.md) 파일을 첨부할 수 있습니다. 빈 줄 = 단락 구분, 마침표/물음표/느낌표 = 문장 분할 기준입니다. 마크다운 헤더와 대괄호 섹션 마커는 영상에 들어가지 않고 그룹 분할 힌트로만 쓰입니다.

# 도입부 오늘은 세종이 장영실에게 곤장 80대를 친 사건을 알아보겠습니다. 조선 최고의 천재 발명가가 왜 사라졌을까요? # 본론 [1장 — 안여 사고] 1442년 봄, 장영실이 만든 임금의 안여가 부서졌습니다. 세종은 격노했고, 장영실은 곤장 80대를 맞았습니다. [2장 — 사라진 천재] 이후 장영실은 역사 기록에서 완전히 사라집니다. 정말 처벌이 그 끝이었을까요?

파싱 규칙

  • # 헤더 — 헤더 줄 자체는 영상에서 제외. "도입" 단어가 포함된 헤더 다음 본문은 자동으로 도입부 그룹으로 분류 (예: # 도입부, ## 영상 도입)
  • [섹션명] 대괄호 마커 — 줄 전체가 [...] 형태일 때만 인식. 영상에서는 제외되고 그룹 경계로만 작동 (예: [1장], [결말])
  • 빈 줄 — 단락 구분. 단락 안의 줄바꿈은 같은 문장의 일부로 합쳐짐
  • 종결부호 . ! ? 。 — 문장 분할 기준. 부호 없이 단락이 끝나면 단락 전체가 한 문장 (긴 문장 → 자동 sub-clip 분할 대상)
  • 따옴표·인용부호 " '「」『』 — 자동 제거 (대화체에서 따옴표가 단독 문장이 되는 사고 방지)
  • 이모지·특수기호 ★ ※ © ™ → ♪ 등 — 자동 제거 (TTS 발음 방지)

권장 작성 흐름

  1. 첫 줄에 # 도입부 헤더 — 후킹 문장 2~3줄 (5~10초 분량)
  2. 본론은 [1장] [2장] ... 처럼 대괄호 마커로 챕터 구분 (옵션)
  3. 한 문장은 마침표로 명확히 끝내기 (긴 문장은 자동 분할되지만 의도된 단락이 더 깔끔)
  4. 대화체는 따옴표 없이 그냥 평문으로 작성
